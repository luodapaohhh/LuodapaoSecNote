# 小程序应用安全测试

小程序（Mini Program）是一种无需下载安装即可使用的应用，运行在微信、支付宝、百度、抖音等超级App内。由于其轻量、便捷的特点，小程序已成为企业服务的重要入口，也因此成为安全测试的重要目标。

## 小程序安全测试概述

小程序安全测试涉及获取小程序包、静态分析、动态分析等多个方面。与原生App不同，小程序运行在容器环境中，其代码通常由JavaScript、WXML、WXSS等构成，并通过宿主App（如微信）的运行时环境执行。

### 测试目标
- 获取小程序源代码和资源文件
- 分析业务逻辑和安全漏洞
- 测试API接口安全性
- 评估数据存储和传输安全

## 1. 获取小程序包

### 1.1 直接搜索与发现
小程序可通过多个平台搜索获取：

| 平台 | 搜索方式 | 特点 |
|------|----------|------|
| **微信** | 微信内搜索、小程序码扫描、分享链接 | 生态最完善，支持最多功能 |
| **支付宝** | 支付宝App内搜索、生活号关联 | 侧重支付和商业服务 |
| **百度** | 百度App搜索、智能小程序 | 依托百度搜索流量 |
| **抖音/头条** | 抖音App内搜索、短视频挂载 | 内容电商和社交属性强 |
| **360小程序** | 360浏览器、360手机助手 | 主要在360生态内 |

**搜索技巧：**
- 使用精确关键词：品牌名+服务类型（如"星巴克点单"）
- 利用分类浏览：各平台都有小程序分类目录
- 关注热门推荐：平台算法推荐的热门小程序
- 企业官网引导：很多企业在官网提供小程序码

### 1.2 小程序包存储位置
小程序首次使用后，包文件会缓存在设备本地：

**微信小程序（Android）：**
```
/data/data/com.tencent.mm/MicroMsg/{user_hash}/appbrand/pkg/
```
- `{user_hash}` 是用户标识的MD5值
- 包文件为`.wxapkg`格式

**微信小程序（iOS）：**
```
/var/mobile/Containers/Data/Application/{app_uuid}/Library/WechatPrivate/{user_id}/WeApp/LocalCache/release/{app_id}/
```
- 需要越狱设备访问

**支付宝小程序：**
```
/data/data/com.eg.android.AlipayGphone/files/小程序缓存/
```

### 1.3 获取包文件的方法

#### 方法一：ADB直接提取（Android）
```bash
# 连接设备
adb devices

# 查找小程序包路径
adb shell "find /data/data/com.tencent.mm -name '*.wxapkg' 2>/dev/null"

# 拉取文件到本地
adb pull /data/data/com.tencent.mm/MicroMsg/xxxxxx/appbrand/pkg/ ~/wxapp_packages/
```

#### 方法二：Root权限文件管理器
- 使用Root Explorer、MT管理器等工具
- 直接导航到缓存目录复制文件

#### 方法三：模拟器环境
- 使用逍遥模拟器、夜神模拟器等
- 在模拟器中运行微信并打开目标小程序
- 从模拟器文件系统中提取包文件

## 2. 静态分析技术

### 2.1 解包工具与步骤
小程序包（.wxapkg）是经过压缩和简单加密的包文件，需要使用专门工具解包：

**常用解包工具：**
- **wxappUnpacker**：最常用的微信小程序解包工具
- **WeChatAppEx**：图形化解包工具
- **unwxapkg**：Python实现的解包脚本

**使用wxappUnpacker解包：**
```bash
# 安装依赖
npm install esprima css-tree cssbeautify vm2 uglify-es

# 解包命令
node wuWxapkg.js 目标小程序.wxapkg -o 输出目录

# 批量解包
for file in *.wxapkg; do node wuWxapkg.js "$file" -o "output_${file%.*}"; done
```

**解包后文件结构：**
```
├── app-service.js    # 小程序逻辑代码（JavaScript）
├── page-frame.html   # 页面框架
├── app-config.json   # 应用配置
├── app.json         # 全局配置
├── pages/           # 页面目录
│   ├── index/
│   │   ├── index.js
│   │   ├── index.wxml
│   │   └── index.wxss
│   └── ...
└── static/          # 静态资源
```

### 2.2 代码分析与审计
解包后的小程序代码可以进行深度分析：

#### 2.2.1 配置文件分析
- **app.json**：全局配置，包含页面路径、窗口样式、网络超时等
- **project.config.json**：项目配置，包含开发者信息

**重点关注：**
```json
{
  "permission": {
    "scope.userLocation": {
      "desc": "你的位置信息将用于小程序位置接口的效果展示"
    }
  },
  "requiredPrivateInfos": ["getLocation", "chooseAddress"]
}
```

#### 2.2.2 JavaScript代码审计
- 搜索敏感API调用：`wx.request`、`wx.uploadFile`、`wx.downloadFile`
- 查找硬编码密钥、token、密码
- 分析鉴权逻辑和会话管理
- 检查输入验证和输出编码

**审计工具：**
```bash
# 使用grep搜索敏感关键词
grep -r "api-key\|secret\|password\|token" .

# 搜索网络请求
grep -r "wx.request\|wx.uploadFile" .

# 搜索存储操作
grep -r "wx.setStorage\|wx.getStorage" .
```

#### 2.2.3 WXML模板分析
- 检查数据绑定和事件处理
- 查找潜在的XSS漏洞
- 分析组件使用和属性传递

#### 2.2.4 WXSS样式分析
- 检查样式注入可能性
- 分析样式覆盖和权限

## 3. 动态分析技术

### 3.1 抓包工具配置
小程序网络请求可通过代理工具捕获：

#### 3.1.1 Burp Suite配置
1. **设置代理监听**：Burp → Proxy → Options → Add (127.0.0.1:8080)
2. **导入CA证书**：
   - 访问 http://burp 下载证书
   - 安装到系统信任存储
3. **设备代理设置**：
   - Android：WLAN → 修改网络 → 高级选项 → 代理手动
   - iOS：Wi-Fi → 配置代理 → 手动

#### 3.1.2 Proxifier配置（处理不走系统代理的应用）
1. 安装Proxifier
2. 添加代理服务器（127.0.0.1:8080）
3. 创建规则：
   - 目标应用程序：微信、支付宝等
   - 动作：Proxy HTTP/HTTPS
4. 启动Proxifier后启动目标App

#### 3.1.3 Fiddler/Charles配置
类似Burp Suite，需要安装CA证书并配置设备代理。

### 3.2 常见抓包问题与解决

#### 问题1：证书验证失败
**现象**：HTTPS请求失败，提示"证书不受信任"

**解决方案：**
```bash
# Android 7+ 需要将CA证书安装到系统证书区
# 需要Root权限
adb root
adb remount
adb push burp-cert-der.crt /system/etc/security/cacerts/
adb shell chmod 644 /system/etc/security/cacerts/burp-cert-der.crt

# 或使用JustTrustMe模块（Xposed/Magisk）
```

#### 问题2：小程序不走代理
**现象**：部分网络请求无法捕获

**解决方案：**
- 使用Proxifier强制代理
- 配置路由器透明代理
- 使用模拟器的网络设置

#### 问题3：请求加密
**现象**：捕获的请求为密文

**解决方案：**
- 逆向分析加密算法
- Hook加密函数（使用Frida、Xposed）
- 查找硬编码密钥

### 3.3 动态调试技术

#### 3.3.1 微信开发者工具调试
1. 安装微信开发者工具
2. 导入解包后的小程序项目
3. 启用调试模式：
   - Sources面板查看和调试JavaScript
   - Console执行命令和查看日志
   - Network监控网络请求
   - Storage查看本地存储

#### 3.3.2 真机调试（Android）
1. 开启USB调试：`adb devices`
2. 端口转发：`adb forward tcp:9222 localabstract:webview_devtools_remote`
3. Chrome访问：`chrome://inspect/#devices`
4. 点击"Inspect"打开DevTools

#### 3.3.3 使用小程序助手工具
- **WeChatDebug**：微信调试工具
- **AlipayDebug**：支付宝调试工具
- **VConsole**：内嵌控制台，需要注入代码

**注入VConsole示例：**
```javascript
// 在app.js开头添加
import VConsole from 'vconsole';
new VConsole();

// 或使用CDN方式
const script = document.createElement('script');
script.src = 'https://cdn.jsdelivr.net/npm/vconsole@latest/dist/vconsole.min.js';
script.onload = function() {
  new VConsole();
};
document.head.appendChild(script);
```

## 4. 安全测试要点

### 4.1 信息泄露
- **源码泄露**：解包获取完整业务逻辑
- **API密钥泄露**：硬编码在代码中的访问密钥
- **敏感配置泄露**：数据库连接字符串、第三方服务密钥

### 4.2 认证与会话管理
- **Token安全**：存储位置、有效期、刷新机制
- **会话固定**：Session ID生成与验证
- **权限绕过**：越权访问其他用户数据

### 4.3 输入验证与输出编码
- **XSS漏洞**：WXML数据绑定、rich-text组件
- **路径遍历**：文件操作API
- **SQL注入**：本地SQLite数据库操作

### 4.4 数据传输安全
- **HTTPS验证**：证书校验、TLS版本
- **数据加密**：敏感数据传输加密
- **防重放攻击**：时间戳、随机数

### 4.5 本地存储安全
- **Storage加密**：wx.setStorage数据是否加密
- **文件权限**：临时文件和缓存文件权限
- **日志安全**：调试信息是否包含敏感数据

## 5. 实战案例：某电商小程序安全测试

### 5.1 测试目标
某品牌电商小程序，主要功能：商品浏览、购物车、订单支付、物流查询。

### 5.2 测试步骤

#### 步骤1：获取小程序包
```bash
# 通过ADB获取
adb pull /data/data/com.tencent.mm/.../pkg/commerce_app.wxapkg

# 解包分析
node wuWxapkg.js commerce_app.wxapkg -o commerce_app
```

#### 步骤2：静态代码审计
发现以下问题：
1. API密钥硬编码在`config.js`中
2. 用户token未设置过期时间
3. 订单ID可预测（顺序递增）

#### 步骤3：动态抓包测试
使用Burp Suite拦截请求：
1. 商品价格篡改：修改`cart/update`请求中的价格参数
2. 越权查看订单：修改`order/detail`中的用户ID参数
3. 重复提交订单：重放`order/create`请求

#### 步骤4：动态调试验证
使用微信开发者工具：
1. 验证前端输入验证可绕过
2. 发现本地存储的token未加密
3. 确认业务逻辑漏洞可利用

### 5.3 漏洞报告
1. **高风险**：订单价格篡改
2. **中风险**：越权访问订单详情
3. **低风险**：API密钥硬编码

## 6. 最佳实践

### 6.1 开发者安全建议
1. **代码混淆与加密**
   - 使用微信官方提供的代码保护功能
   - 对敏感逻辑进行服务器端验证

2. **密钥管理**
   - 避免硬编码API密钥
   - 使用临时token和定期刷新
   - 密钥存储在服务器端

3. **输入验证**
   - 客户端和服务端双重验证
   - 使用白名单验证机制
   - 对输出进行适当编码

4. **会话安全**
   - 使用HTTPS传输
   - 设置合理的token过期时间
   - 实现安全的退出机制

### 6.2 测试人员建议
1. **全面覆盖测试点**
   - 静态分析 + 动态测试结合
   - 关注业务逻辑漏洞
   - 测试边界条件和异常情况

2. **工具链准备**
   - 解包工具：wxappUnpacker
   - 抓包工具：Burp Suite + Proxifier
   - 调试工具：微信开发者工具 + Chrome DevTools

3. **环境搭建**
   - 准备Root过的Android测试机
   - 配置好代理和证书的测试环境
   - 模拟真实用户场景测试

## 总结

小程序安全测试需要结合静态分析和动态测试，重点关注：
1. **代码安全性**：源码保护、密钥管理、输入验证
2. **通信安全**：HTTPS实施、数据加密、防重放
3. **本地安全**：存储加密、文件权限、日志管理
4. **业务逻辑**：权限控制、流程验证、异常处理

随着小程序生态的不断发展，安全测试方法和工具也在持续演进。测试人员需要保持学习，及时掌握最新的测试技术和漏洞模式。
