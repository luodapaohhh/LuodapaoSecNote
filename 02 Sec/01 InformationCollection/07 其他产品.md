# 其他产品信息收集

除传统Web应用、主机服务器外，企业信息可能分散在网盘、代码仓库、敏感目录及网络空间搜索引擎中。本节系统介绍这些渠道的信息收集方法与实操技巧。

## 网盘文件搜索

网盘存储了大量企业文档、代码、配置文件，常因权限设置不当而公开暴露敏感信息。

### 主要网盘平台

1. **百度网盘** (pan.baidu.com)
   - 中国最常用网盘，存储大量企业文档
   - 支持文件名搜索，但公开分享链接需特定格式

2. **Google Drive** (drive.google.com)
   - 国际常用，企业协作文档常存储于此
   - 可通过Google搜索语法定位公开文件

3. **其他平台**：
   - **腾讯微云**：微信、QQ关联文件
   - **坚果云**：企业文件同步
   - **OneDrive**：Office 365企业套件
   - **Dropbox**：国际团队协作

### 搜索技巧与方法

#### 基础搜索语法
```bash
# Google搜索特定网盘文件
site:pan.baidu.com "企业名称" 招标
site:drive.google.com "company_name" password
site:*.yunpan.cn "内部" ext:docx
```

#### 高级搜索策略

**1. 文件类型限定**：
- `filetype:pdf "机密"` - 搜索PDF文档
- `ext:docx "内部会议纪要"` - Word文档
- `ext:xlsx "员工信息"` - Excel表格
- `ext:sql "数据库备份"` - SQL文件
- `ext:zip "源码"` - 压缩包

**2. 关键词组合**：
- **企业信息**：`"企业名" 员工通讯录 电话`
- **招标信息**：`"项目名称" 招标文件 预算`
- **源码泄露**：`"github" node_modules backup`
- **配置信息**：`config.json database password`
- **API密钥**：`api_key secret token`

**3. 时间范围限定**：
- `after:2023-01-01 before:2023-12-31` - 特定年份
- 最近泄露的文件价值更高

## GitHub信息收集

GitHub作为最大代码托管平台，常泄露源码、配置、密钥等敏感信息。

### 信息收集维度

#### 1. 直接搜索目标组织
- `org:companyname` - 搜索组织下所有仓库
- `user:username` - 特定用户仓库
- `repo:owner/repo` - 指定仓库

#### 2. 敏感信息搜索
```bash
# API密钥与令牌
"api_key" "company" 
"secret" "password"
"access_token" "AKIA" (AWS密钥)
"sk_live_" (Stripe密钥)

# 配置文件
"config.json" "database"
".env" "DB_PASSWORD"
"application.properties" 

# 源码注释
"TODO" "FIXME" "HACK" "BUG"
"漏洞" "安全" "测试"
```

#### 3. 员工信息收集
- **员工GitHub账号**：通过企业邮箱反查
- **提交历史**：`git log` 中的姓名、邮箱
- **协作网络**：关注、星标、fork关系

### GitHub高级搜索语法

#### 代码搜索
```bash
# 按语言筛选
language:python "password"
language:javascript "apiKey"

# 按文件路径
path:.env "DATABASE_URL"
path:config/ "secret"

# 按文件大小
size:>1000 "credentials"  # 大于1KB的文件
```

#### 时间范围
```bash
# 按提交时间
created:>2023-01-01
pushed:>2024-01-01

# 最近泄露
created:>2024-04-01  # 最近创建
```

#### 仓库属性
```bash
# 按星标数
stars:>100  # 热门项目
forks:>50   # 被fork次数

# 按是否fork
fork:true   # 仅搜索fork的仓库
fork:false  # 仅搜索原创仓库
```

#### 4. 自建监控系统
```python
# 简化的GitHub API监控脚本
import requests
import time

def monitor_github(keywords, interval=300):
    """监控GitHub代码提交"""
    url = "https://api.github.com/search/code"
    headers = {"Authorization": "token YOUR_TOKEN"}
    
    for keyword in keywords:
        params = {"q": keyword, "sort": "indexed", "order": "desc"}
        response = requests.get(url, headers=headers, params=params)
        # 处理结果...
        print(f"搜索 {keyword}: {len(response.json()['items'])} 个结果")
    
    time.sleep(interval)

# 使用示例
monitor_github(["api_key", "password", "example.com"])
```

### 企业GitHub信息收集流程

1. **目标识别**：
   - 确定企业官方GitHub组织
   - 收集员工个人账号（通过领英、邮箱等）

2. **初步搜索**：

   - 分析仓库提交历史
   - 查看issue、pull request讨论
   - 检查wiki、项目文档

4. **信息关联**：
   - 将找到的邮箱、账号与其他渠道关联
   - 分析代码中的内部域名、API地址
   - 提取技术栈信息（package.json, requirements.txt）

5. **持续监控**：
   - 设置关键词告警
   - 定期复查高价值仓库
   - 跟踪新员工账号活动

### 防御措施
1. **代码扫描**：部署pre-commit钩子，检测敏感信息
2. **员工培训**：禁止提交密钥、配置文件
3. **仓库审计**：定期扫描历史提交，清理敏感信息
4. **访问控制**：私有仓库严格权限管理

### 扫描工具与方法

#### 1. 字典扫描工具

**dirsearch** (https://github.com/maurosoria/dirsearch)：
```bash

# 扩展选项
```

```bash

```

**ffuf** (高度可配置)：

#### 2. 智能爬虫

**Burp Suite**：
- **Spider**：自动爬取应用链接
- **Content Discovery**：基于字典的智能扫描
- **Sequencer**：分析会话标识符

**定制爬虫脚本**：
```python
import requests
from bs4 import BeautifulSoup
import re

def crawl_sensitive_paths(url, wordlist):
    """爬取目标并检测敏感路径"""
    sensitive_patterns = [
        r'(admin|login|dashboard)',
        r'\.(bak|zip|tar|gz)$',
        r'(config|setting)\.(php|yml|json)',
        r'\.git/',
        r'(password|secret|key)'
    ]
    
    # 爬取页面
    response = requests.get(url)
    soup = BeautifulSoup(response.text, 'html.parser')
    
    # 提取所有链接
    links = [a.get('href') for a in soup.find_all('a', href=True)]
    
    # 检测敏感路径
    for link in links:
        for pattern in sensitive_patterns:
            if re.search(pattern, link, re.IGNORECASE):
                print(f"[敏感路径] {link}")
                break
```

### 证书资产收集

SSL/TLS证书包含域名、组织、邮箱等信息，是资产发现重要来源。

#### 1. 证书透明日志（CT Log）
- 所有公开SSL证书都被记录在CT Log
- 可通过crt.sh、Google透明证书搜索查询

**crt.sh查询**：
```sql
-- 查询example.com的所有证书
SELECT * FROM certificate WHERE NAME_VALUE LIKE '%.example.com%'
```

**实操命令**：
```bash
# 使用crt.sh API
curl -s "https://crt.sh/?q=%.example.com&output=json" | jq '.[].name_value'

# 提取所有子域名
curl -s "https://crt.sh/?q=%.example.com&output=json" | jq -r '.[].name_value' | sort -u
```

#### 2. Censys证书搜索
```
443.https.tls.certificate.parsed.subject.common_name: "example.com"
```

**搜索示例**：
- 查找同一组织所有证书：`parsed.subject.organization: "Company Name"`
- 查找自签名证书：`tags: self-signed`
- 查找过期证书：`parsed.validity.end: <2024-01-01`

#### 3. 工具自动化
**certgraph**：绘制证书关联图
```bash
certgraph example.com --output graph.png
```

### ICO图标资产

网站favicon.ico具有唯一性，可通过哈希值在Shodan、FOFA中搜索相同图标站点。

#### 1. 图标哈希计算
```python
import mmh3
import requests
import codecs

def get_favicon_hash(url):
    """计算favicon.ico的MurmurHash3值"""
    try:
        response = requests.get(f"{url}/favicon.ico")
        if response.status_code == 200:
            favicon = codecs.encode(response.content, "base64")
            hash_value = mmh3.hash(favicon)
            return hash_value
    except:
        return None

# 使用示例
hash_val = get_favicon_hash("https://example.com")
print(f"Favicon哈希: {hash_val}")
```

#### 2. Shodan搜索
```
http.favicon.hash: -1234567890
```

#### 3. FOFA搜索
```
icon_hash="-1234567890"
```

#### 4. 实际应用
1. 获取目标主站favicon哈希
2. 在Shodan/FOFA搜索相同哈希
3. 发现使用相同系统的其他站点（可能为测试、预发布环境）

### 集成工作流示例

将网络空间搜索整合到标准信息收集流程：

```bash
#!/bin/bash
# 自动化资产发现脚本示例

TARGET="example.com"

# 1. 证书资产
echo "=== 证书资产 ==="
curl -s "https://crt.sh/?q=%.${TARGET}&output=json" | jq -r '.[].name_value' | sort -u > certificates.txt

# 2. 子域名发现（结合证书）
cat certificates.txt | grep "${TARGET}" | sort -u > subdomains.txt

# 3. 端口扫描（对发现的主机）
for domain in $(head -20 subdomains.txt); do
    echo "扫描 $domain"
    nmap -sS -T4 -p 80,443,8080,8443 $domain >> nmap_scan.txt
done

# 4. 敏感路径扫描
for domain in $(head -10 subdomains.txt); do
    echo "扫描 $domain 敏感路径"
    python3 dirsearch.py -u https://$domain -e php,html,json -t 30
done

# 5. 结果汇总
echo "=== 资产汇总 ==="
echo "子域名数量: $(wc -l < subdomains.txt)"
echo "开放端口: $(grep 'open' nmap_scan.txt | wc -l)"
```

## 总结与最佳实践

### 1. 工具链整合
- **网盘搜索**：Google高级语法 + 专用搜索平台
- **GitHub监控**：自动化脚本 + 告警系统
- **目录扫描**：dirsearch/gobuster + 定制字典
- **网络空间**：Shodan/FOFA + 证书/ICO查询

### 2. 信息验证原则
- 多渠道交叉验证（如GitHub找到的域名，用证书搜索确认）
- 区分生产/测试环境资产
- 评估信息时效性（最近更新 vs 历史遗留）

### 3. 合法合规边界
- 仅收集公开可用信息
- 遵守目标robots.txt协议
- 不进行未授权漏洞扫描
- 企业测试需书面授权

### 4. 持续监控策略
1. **定期任务**：每周/每月重复关键搜索
2. **实时告警**：设置GitHub、证书监控
3. **趋势分析**：跟踪资产变化，发现新暴露面

通过系统化的"其他产品"信息收集，可以构建更完整的企业数字资产画像，为后续安全评估提供坚实基础。
