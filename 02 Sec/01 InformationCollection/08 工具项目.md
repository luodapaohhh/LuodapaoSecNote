## 核心工具详细文档

### 1. Nmap (Network Mapper)

#### 分步教程

**安装说明：**

```bash
# Ubuntu/Debian
sudo apt update
sudo apt install nmap

# CentOS/RHEL  
sudo yum install nmap

# macOS
brew install nmap

# Windows
# 下载安装包：https://nmap.org/download.html
```

**基础扫描命令：**

```bash
# 1. TCP SYN扫描（半开扫描，最常用）
sudo nmap -sS 192.168.1.1

# 2. TCP Connect扫描（全连接，不需要root）
nmap -sT 192.168.1.1

# 3. UDP扫描
sudo nmap -sU 192.168.1.1

# 4. 指定端口范围
nmap -p 1-1000 192.168.1.1            # 扫描1-1000端口
nmap -p 80,443,8080,8443 192.168.1.1  # 扫描特定端口
nmap -p- 192.168.1.1                  # 扫描所有端口（1-65535）

# 5. 服务版本检测
nmap -sV 192.168.1.1

# 6. 操作系统检测
sudo nmap -O 192.168.1.1

# 7. 使用NSE脚本进行漏洞检测
nmap --script vuln 192.168.1.1

# 8. 保存扫描结果
nmap -oN scan.txt 192.168.1.1         # 普通文本格式
nmap -oX scan.xml 192.168.1.1         # XML格式
nmap -oG scan.gnmap 192.168.1.1       # Grepable格式
```

**高级扫描技巧：**

```bash
# 绕过防火墙/IDS
nmap -f --mtu 8 192.168.1.1           # 使用分片
nmap --data-length 100 192.168.1.1    # 添加随机数据长度
nmap --source-port 53 192.168.1.1     # 使用特定源端口（如DNS）
nmap -D RND:10 192.168.1.1            # 使用诱饵IP（10个随机IP）

# 时序和性能控制
nmap -T0 192.168.1.1                  # 最慢（避免被检测）
nmap -T1 192.168.1.1                  # 较慢
nmap -T2 192.168.1.1                  # 正常
nmap -T3 192.168.1.1                  # 较快（默认）
nmap -T4 192.168.1.1                  # 快速
nmap -T5 192.168.1.1                  # 最快（可能丢包）

# 网络发现（不进行端口扫描）
nmap -sn 192.168.1.0/24              # Ping扫描，发现存活主机
nmap -PR 192.168.1.0/24              # ARP扫描（本地网络）
```

#### 实际案例研究

**场景：** 某企业外部网络安全评估，授权范围：203.0.113.0/24

**扫描过程：**

```bash
# 阶段1：快速发现
nmap -sn 203.0.113.0/24 -oN ping_scan.txt
nmap --top-ports 100 203.0.113.0/24 -oN top_ports.txt

# 提取存活主机
grep -oP 'Nmap scan report for \K[^ ]*' ping_scan.txt > live_hosts.txt

# 阶段2：详细扫描
for host in $(cat live_hosts.txt); do
    echo "扫描主机: $host"
    nmap -sV -O -p- $host -oN detailed_$host.txt
done

# 阶段3：漏洞扫描
nmap --script vuln -iL live_hosts.txt -oN vuln_scan.txt
```

**发现结果：**
1. **服务器1 (203.0.113.10)**：开放80(Apache 2.4.41)、443(HTTPS)、22(OpenSSH 7.6p1)
2. **服务器2 (203.0.113.20)**：开放21(ProFTPD 1.3.5e)、25(Postfix)、110/143(Dovecot)
3. **服务器3 (203.0.113.30)**：开放3389(Windows RDP)、5985/5986(WinRM)

**安全风险：**
- ProFTPD存在CVE-2021-XXXX漏洞
- RDP服务存在BlueKeep风险
- Web服务器泄露版本信息

#### 最佳实践

1. **法律合规：**
   - 仅在被授权范围内扫描
   - 避免在业务高峰时段扫描
   - 记录所有操作日志

2. **技术优化：**
   - 小规模网络使用Nmap，大规模使用Masscan+Nmap组合
   - 控制扫描速率：`--max-rate 100`（100包/秒）
   - 使用随机延迟：`--scan-delay 5s`
   - 分阶段扫描：先快速发现，再深度扫描

3. **结果分析：**
   - 验证扫描结果，排除误报
   - 关联分析不同工具的扫描结果
   - 按风险等级对发现的问题排序

4. **常见误区：**
   - 避免过度扫描（使用最激进参数）
   - 不要完全信任自动化工具结果
   - 结合业务重要性评估风险

### 2. FOFA 网络空间搜索引擎

#### 分步教程

**注册与使用：**

1. **访问官网：** https://fofa.info
2. **注册账号：** 使用邮箱注册，免费版有基本查询额度
3. **获取API密钥：** 个人中心 → API接口 → 生成API Key

**基础搜索语法：**

```bash
# 应用组件
app="Apache"
app="Jenkins"
app="GitLab"

# 页面标题
title="管理员登录"
title="登录"
title="后台管理"

# HTTP头部
header="thinkphp"
header="php"
header="spring"

# 页面内容
body="验证码错误"
body="用户名或密码错误"
body="Copyright"

# 网络信息
ip="1.1.1.1"
domain="example.com"
port="8080"
country="CN"
city="beijing"
```

**命令行工具使用：**

```bash
# 安装fofa-cli
pip install fofa

# 配置API密钥
export FOFA_EMAIL="your_email@example.com"
export FOFA_KEY="your_api_key"

# 基础搜索
fofa --query 'app="Apache"'

# 指定字段输出
fofa --query 'domain="example.com"' --fields ip,port,title

# 保存结果
fofa --query 'app="Jenkins"' --output jenkins_hosts.txt
```

**API调用示例：**

```python
import requests
import base64

email = "your_email"
key = "your_api_key"
query = 'domain="example.com"'

# FOFA API要求查询语句base64编码
query_base64 = base64.b64encode(query.encode()).decode()

url = f"https://fofa.info/api/v1/search/all?email={email}&key={key}&qbase64={query_base64}&size=1000"
response = requests.get(url)
results = response.json()

for result in results['results']:
    print(f"IP: {result[0]}, Port: {result[1]}, Title: {result[2]}")
```

#### 实际案例研究

**场景：** 发现某企业暴露的测试环境和内部系统

**搜索策略：**

```bash
# 1. 搜索企业主域名
domain="example.com"

# 2. 搜索相关子域名
host="example.com"

# 3. 搜索特定技术栈
app="Spring Boot" && domain="example.com"

# 4. 搜索管理后台
title="登录" && domain="example.com"

# 5. 搜索测试环境
title="测试" || title="dev" || title="staging"

# 6. 搜索监控面板
app="Grafana" || app="Kibana" || title="Dashboard"

# 7. 搜索代码仓库
app="GitLab" || app="Gitea" || port="3000"

# 8. 搜索数据库服务
port="3306" || port="5432" || port="6379"
```

**实战发现：**

1. **测试环境泄露：** `dev.example.com` 暴露Spring Boot Actuator端点
2. **监控面板未授权访问：** `monitor.example.com:3000` Grafana无需认证
3. **数据库外网可访问：** `db.example.com:3306` MySQL允许远程连接
4. **内部文档系统：** `wiki.example.com` Confluence可见内部技术文档
5. **CI/CD系统：** `jenkins.example.com` Jenkins控制台可访问

**风险分析：**
- 测试环境通常安全防护较弱
- 监控面板可能包含系统指标和日志
- 数据库直接暴露可能导致数据泄露
- 内部文档泄露技术架构和配置信息

#### 最佳实践

1. **搜索技巧：**
   - 使用`&&`（AND）和`||`（OR）组合条件
   - 从宽到窄逐步缩小范围
   - 尝试多种关键词变体
   - 关注证书透明度日志发现的域名

2. **企业资产发现流程：**
   ```
   主域名搜索 → 子域名发现 → 技术栈识别 → 敏感系统定位 → 风险评估
   ```

3. **数据整理：**
   - 去重：`sort -u`
   - 分类：按服务类型、风险等级分类
   - 验证：手动访问确认资产有效性

4. **合规使用：**
   - 仅用于授权测试
   - 不进行未授权的漏洞扫描
   - 尊重robots.txt协议

### 3. OneForAll 子域名收集工具

#### 分步教程

**安装部署：**

```bash
# 1. 克隆项目
git clone https://github.com/shmilylty/OneForAll.git
cd OneForAll

# 2. 安装依赖（Python 3.8+）
pip3 install -r requirements.txt

# 3. 配置API密钥（可选但推荐）
# 编辑config/api.py，添加各种服务的API密钥
# 如SecurityTrails、VirusTotal、Shodan等
```

**基础使用：**

```bash
# 基本扫描
python3 oneforall.py --target example.com run

# 指定输出目录
python3 oneforall.py --target example.com --path ./results run

# 使用代理
python3 oneforall.py --target example.com --proxy socks5://127.0.0.1:1080 run

# 仅使用部分模块
python3 oneforall.py --target example.com --brute False run  # 关闭爆破
python3 oneforall.py --target example.com --req False run    # 关闭HTTP请求验证
```

**高级功能：**

```bash
# 批量扫描
python3 oneforall.py --targets domains.txt run

# 从文件读取目标
python3 oneforall.py --target_file domains.txt run

# 结果去重和整理
python3 oneforall.py --target example.com --format csv --alive True run

# 使用自定义字典
python3 oneforall.py --target example.com --wordlist subdomains.txt run
```

**结果处理：**

```bash
# 查看结果
ls -la results/
cat results/example.com.csv

# 提取存活的子域名
grep ",alive," results/example.com.csv | cut -d, -f1

# 转换为其他格式
python3 oneforall.py --target example.com --format json run
```

#### 实际案例研究

**场景：** 全面收集某电商企业的子域名资产

**实施步骤：**

```bash
# 1. 完整扫描
python3 oneforall.py --target yougou.com --alive True run

# 2. 分析结果
cat results/yougou.com.csv | column -t -s,

# 发现的关键子域名：
# - www.yougou.com          # 主站
# - api.yougou.com          # API接口
# - admin.yougou.com        # 管理后台
# - m.yougou.com           # 移动端
# - cdn.yougou.com         # CDN服务
# - static.yougou.com      # 静态资源
# - dev.yougou.com         # 开发环境
# - test.yougou.com        # 测试环境
# - jenkins.yougou.com     # CI/CD
# - grafana.yougou.com     # 监控面板

# 3. 存活验证
for domain in $(grep ",alive," results/yougou.com.csv | cut -d, -f1); do
    echo "检查: $domain"
    curl -I "https://$domain" 2>/dev/null | head -1
done

# 4. 分类整理
# 业务系统：www, api, admin, m
# 支撑系统：cdn, static
# 开发测试：dev, test
# 内部系统：jenkins, grafana
```

**发现的安全问题：**

1. **测试环境暴露：** dev.yougou.com可直接访问，包含测试数据
2. **监控面板未授权：** grafana.yougou.com无需认证
3. **管理后台暴露：** admin.yougou.com使用弱密码可爆破
4. **API接口未鉴权：** api.yougou.com部分端点无需token

#### 最佳实践

1. **配置优化：**
   - 申请必要的API密钥提高发现率
   - 根据目标调整超时和重试参数
   - 使用代理池避免IP被封

2. **扫描策略：**
   - 先使用passive模块（不发送请求）
   - 再使用active模块（发送少量请求）
   - 最后使用爆破模块（谨慎使用）

3. **结果验证：**
   - 自动验证：启用`--alive True`
   - 手动验证：关键域名手动访问确认
   - 去重处理：合并多个来源的结果

4. **企业级应用：**
   - 定期扫描监控资产变化
   - 集成到资产管理系统
   - 设置告警机制（发现新域名）

### 4. ARL灯塔（Asset Reconnaissance Lighthouse）

#### 分步教程

**Docker部署：**

```bash
# 1. 安装Docker和Docker Compose
sudo apt install docker.io docker-compose

# 2. 克隆ARL仓库
git clone https://github.com/TophantTechnology/ARL.git
cd ARL

# 3. 修改配置文件（可选）
vim docker/config-docker.yaml

# 4. 启动服务
docker-compose up -d

# 5. 访问Web界面
# 地址：http://localhost:5003
# 默认账号：admin/arlpass
```

**基础使用流程：**

1. **添加扫描任务：**
   - 目标：单个域名、IP或IP段
   - 扫描类型：域名扫描、端口扫描、URL扫描
   - 高级选项：端口设置、线程数、深度等

2. **查看扫描结果：**
   - 资产列表：域名、IP、端口、服务
   - 漏洞检测：弱口令、常见漏洞
   - 报告导出：HTML、PDF、Excel

3. **API调用：**
   ```python
   import requests
   
   # ARL API地址
   url = "http://localhost:5003/api/"
   
   # 登录获取token
   login_data = {"username": "admin", "password": "arlpass"}
   response = requests.post(url + "user/login", json=login_data)
   token = response.json()["data"]["token"]
   
   # 创建扫描任务
   headers = {"Authorization": f"Bearer {token}"}
   task_data = {
       "target": "example.com",
       "domain_brute": True,
       "alt_dns": True,
       "port_scan": True,
       "service_detection": True
   }
   response = requests.post(url + "task", json=task_data, headers=headers)
   task_id = response.json()["data"]["task_id"]
   ```

**常用扫描配置：**

```yaml
# 域名扫描
domain_brute: true        # 域名爆破
dns_query: true           # DNS查询
certificate: true         # 证书查询
subdomain_takeover: true  # 子域名接管检测

# 端口扫描
port_scan: true           # 端口扫描
port_list: "top100"       # 端口列表：top100, top1000, full
service_detection: true   # 服务识别
os_detection: true        # 操作系统识别

# Web扫描
web_info: true            # Web信息收集
web_vuln: true            # Web漏洞扫描
dir_scan: true            # 目录扫描
url_extract: true         # URL提取
```

#### 实际案例研究

**场景：** 企业内网资产自动化发现与监控

**部署架构：**
```
ARL主节点（Docker） + 多个扫描节点 + 定时任务 + 告警通知
```

**实施流程：**

1. **初始化扫描：**
   ```bash
   # 扫描整个C段
   192.168.1.0/24
   
   # 扫描所有业务域名
   *.example.com
   
   # 定期扫描策略：每周一次全量，每天增量
   ```

2. **资产发现结果：**
   - **网络设备：** 路由器、交换机、防火墙（192.168.1.1）
   - **服务器：** Web服务器、数据库、文件服务器
   - **办公设备：** 打印机、摄像头、门禁系统
   - **特殊设备：** 工控设备、医疗设备、IoT设备

3. **风险识别：**
   - **弱口令：** 多台设备使用默认密码
   - **未授权访问：** Redis、MongoDB无密码
   - **漏洞设备：** 旧版本Apache、Tomcat
   - **暴露服务：** SMB、RDP外网可访问

4. **自动化响应：**
   - 发现新资产自动入库
   - 高风险漏洞即时告警
   - 生成修复工单

#### 最佳实践

1. **部署建议：**
   - 生产环境使用独立数据库
   - 配置备份和恢复机制
   - 设置访问控制和审计日志

2. **扫描优化：**
   - 根据网络环境调整并发数
   - 避开业务高峰时段
   - 使用白名单避免误扫关键设备

3. **企业集成：**
   - 与CMDB同步资产数据
   - 集成SIEM和SOC平台
   - 对接工单系统自动派单

4. **持续运营：**
   - 定期更新漏洞规则库
   - 优化扫描策略提高效率
   - 培训运营人员分析报告

### 5. Wappalyzer 技术栈识别

#### 分步教程

**浏览器扩展安装：**

1. **Chrome浏览器：**
   - 访问 Chrome Web Store
   - 搜索 "Wappalyzer"
   - 点击 "添加到 Chrome"

2. **Firefox浏览器：**
   - 访问 Firefox Add-ons
   - 搜索 "Wappalyzer"
   - 点击 "添加到 Firefox"

3. **Edge浏览器：**

### 6. Burp Suite Web安全测试

#### 分步教程

**安装与配置：**

1. **下载安装：**
   - 社区版：https://portswigger.net/burp/communitydownload
   - 专业版：需要许可证

2. **初始配置：**
   - 启动Burp Suite
   - 创建临时项目或打开已有项目
   - 配置代理监听器（Proxy → Options）

3. **浏览器代理设置：**
   - 浏览器设置 → 网络设置 → 手动代理配置
   - HTTP代理：127.0.0.1 端口：8080
   - 安装Burp CA证书（访问 http://burp 下载）

**核心模块使用：**

1. **Proxy（代理）：**
   - 拦截HTTP/HTTPS请求和响应
   - 可修改请求参数、头部、内容
   - 支持自动转发、禁用拦截等模式

2. **Repeater（重放器）：**
   - 手动修改和重发单个请求
   - 观察不同参数的效果
   - 测试漏洞验证和利用

3. **Intruder（入侵者）：**
   - 自动化参数爆破和模糊测试
   - 四种攻击类型：Sniper、Battering ram、Pitchfork、Cluster bomb
   - 支持自定义Payload和编码

4. **Scanner（扫描器）：**
   - 自动Web漏洞扫描（专业版）
   - 被动扫描和主动扫描
   - 生成详细漏洞报告

5. **Extender（扩展）：**
   - 安装BApp Store中的扩展
   - 自定义扩展开发（Java/Python）

**基础测试流程：**

```bash
1. 配置浏览器代理
2. 访问目标网站
3. 拦截登录请求
4. 发送到Repeater修改参数
5. 使用Intruder爆破弱口令
6. 分析响应寻找漏洞
```

### 7. dirsearch 目录扫描工具

#### 分步教程

**安装部署：**

```bash
# 1. 克隆项目
git clone https://github.com/maurosoria/dirsearch.git
cd dirsearch

# 2. 安装依赖
pip3 install -r requirements.txt

# 3. 查看帮助
python3 dirsearch.py -h
```

**基础扫描：**

```bash
# 基本扫描
python3 dirsearch.py -u https://example.com -e php,html,json

# 指定字典
python3 dirsearch.py -u https://example.com -w /path/to/wordlist.txt

# 递归扫描
python3 dirsearch.py -u https://example.com -e php --recursive

# 扩展选项
python3 dirsearch.py -u https://example.com \
  -t 50 \                    # 线程数
  --timeout 10 \            # 超时时间
  --random-agents \         # 随机User-Agent
  --proxy http://127.0.0.1:8080  # 使用代理
```

**结果处理：**

```bash
# 保存结果到文件
python3 dirsearch.py -u https://example.com -o report.txt

# JSON格式输出
python3 dirsearch.py -u https://example.com --json-report report.json

# 只显示特定状态码
python3 dirsearch.py -u https://example.com --status-codes 200,301,302

# 排除特定状态码
python3 dirsearch.py -u https://example.com --exclude-status-codes 404,403
```

**字典选择策略：**

```bash
# 通用字典
-w /usr/share/wordlists/dirb/common.txt
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

# CMS专用字典
-w /path/to/wordpress.txt
-w /path/to/joomla.txt

# 扩展名字典
-e php,asp,aspx,jsp,html,json,xml
-e bak,zip,tar,gz,sql
```

```bash
#!/bin/bash
# auto_recon.sh - 自动化信息收集脚本

TARGET=$1
OUTPUT_DIR="./recon_$(date +%Y%m%d_%H%M%S)"

mkdir -p $OUTPUT_DIR

echo "[*] 开始收集: $TARGET"
echo "[*] 输出目录: $OUTPUT_DIR"

# 1. 子域名发现
echo "[1/5] 子域名枚举..."
python3 oneforall.py --target $TARGET --path $OUTPUT_DIR/oneforall run

# 2. 端口扫描
echo "[2/5] 端口扫描..."
masscan -p1-65535 --rate=1000 -oX $OUTPUT_DIR/masscan.xml $TARGET

# 3. Web资产识别
echo "[3/5] Web资产识别..."
cat $OUTPUT_DIR/oneforall/$TARGET.csv | grep ",alive," | cut -d, -f1 | while read domain; do
    whatweb $domain >> $OUTPUT_DIR/whatweb.txt
done

# 4. 目录扫描
echo "[4/5] 目录扫描..."
cat $OUTPUT_DIR/oneforall/$TARGET.csv | grep ",alive," | cut -d, -f1 | head -10 | while read domain; do
    python3 dirsearch.py -u https://$domain -e php,html -t 20 >> $OUTPUT_DIR/dirsearch.txt
done

# 5. 生成报告
echo "[5/5] 生成报告..."
echo "=== 信息收集报告 ===" > $OUTPUT_DIR/report.txt
echo "目标: $TARGET" >> $OUTPUT_DIR/report.txt
echo "时间: $(date)" >> $OUTPUT_DIR/report.txt
echo "子域名数量: $(grep -c ",alive," $OUTPUT_DIR/oneforall/$TARGET.csv)" >> $OUTPUT_DIR/report.txt
echo "开放端口: $(grep -c 'state="open"' $OUTPUT_DIR/masscan.xml)" >> $OUTPUT_DIR/report.txt

echo "[+] 收集完成!"
```
